Data Report Automation
Lyft - Summer 2019

Overview

Lyft's Regulatory Reporting team is in charge of building and sending off many data reports of varying sizes and degrees of intricacy. Some of the smaller, more monotonous reports need to frequently be sent out to regulators, but the process of gathering data, formatting, and sending such reports is the same every time. My task was to create a template that my team could use to turn their manual processes for smaller reports into fully automated processes to gain back work time. This was an individual project that I worked on from July to August 2019. I received help from a Lyft software engineer, Sam Reese, and I worked cross functionally with other engineers and an external secure digital storage company for various clearances and API needs, respectively.

The Code

I designed a Python program that contains all 3 parts of the report process:


1. Pull raw data from Lyft’s data tables using Hive.


2. Validate and format the data in Pandas.


3. Send the data report using Python and the secure digital storage company’s API to the report's storage location for regulators.


In order to actually automate the whole process, I used Apache Airflow and a directed acyclic graph (DAG) in Python to build the steps of the workflow and schedule recurring runs of the Python program. I went through the design process for one report, and then built a general template and wrote documentation in order for my team to continue my work and automate more reports in the future.
